## 📌 현재 완료한 기술들

### 프론트엔드
**Next.js 14** - React 프레임워크로 블로그의 모든 페이지를 만들었어요.
- App Router로 폴더 구조만으로 라우팅 자동 설정
- 서버/클라이언트 컴포넌트 구분

**React 18+** - UI 컴포넌트와 상태 관리
- useState로 동적 상태 처리
- useEffect로 부수 효과 관리

**TypeScript** - 타입 안전성으로 버그 줄이기
- 인터페이스로 데이터 구조 명시
- 제네릭으로 재사용 가능한 컴포넌트 작성

**Tailwind CSS** - 빠른 스타일링
- 유틸리티 클래스로 직관적인 디자인
- 커스터마이징으로 개성있는 색상 팔레트

**MDX** - 마크다운에 React 컴포넌트 삽입
- 블로그 포스트를 동적으로 렌더링
- 코드 예제를 인터랙티브하게 표현 가능

### 백엔드 & DevOps
**Git & GitHub** - 버전 관리와 협업 기초
- `git add`, `commit`, `push`로 코드 관리
- GitHub에 원격 저장소 유지

**Vercel** - Next.js 배포의 최고의 선택
- GitHub 연동으로 자동 배포
- 무료 티어로도 충분한 성능

**Node.js** - 자바스크립트 백엔드 런타임
- npm으로 패키지 관리
- 스크립트 자동화 (sync-blog.ts)

### 개발 도구
**Claude Code** - AI 코딩 도우미 (활발히 사용 중)
- 복잡한 로직도 빠르게 구현
- 코드 리뷰와 최적화 제안

**VS Code** - 가볍고 강력한 에디터
- 확장 프로그램으로 기능 확장
- Git 통합으로 편한 관리

## 🚀 근기간 학습 목표 (1-2개월)

### 1️⃣ TypeScript 심화 | 2-3주
**목표:** 제네릭, 고급 타입, 유틸리티 타입 마스터

```typescript
// 제네릭으로 재사용 가능한 함수
function getFirst<T>(arr: T[]): T {
  return arr[0];
}

// 유틸리티 타입으로 타입 변환
type Readonly<T> = {
  readonly [K in keyof T]: T[K];
};
```

**이유:** 타입 안전성으로 버그를 사전에 방지하고, 코드 품질 향상

**블로그에 적용:** 포스트 메타데이터 타입을 더 정확하게 정의

### 2️⃣ Python 기초 | 3-4주
**목표:** 파이썬 문법, 리스트/딕셔너리, 함수, 모듈

**왜 배워야 할까?**
- AI/ML 작업의 표준 언어
- 데이터 처리와 자동화 스크립트
- Obsidian 동기화 스크립트를 더 강력하게

**프로젝트:** Obsidian 노트를 자동으로 분류하는 스크립트 작성

### 3️⃣ Cursor (AI 코드 에디터) | 1주
**목표:** Claude Code와 비교, 장단점 파악

**궁금한 점:**
- Claude Code와 어떻게 다를까?
- 어떤 상황에서 더 효율적일까?
- 두 도구를 함께 사용할 수 있을까?

## 🔥 중기 학습 목표 (2-3개월)

**목표:** 객체지향 프로그래밍, 클래스, 상속, 인터페이스

**활용:** 블로그에 댓글 시스템, 사용자 인증 추가

### AI & 머신러닝 기초 | 4-6주
**목표:** 기본 개념, Claude API 활용, 프롬프트 엔지니어링

**블로그에 추가할 AI 기능:**
- 블로그 추천 시스템 (읽은 포스트 기반)
- 자동 요약 기능 (긴 포스트 요약)
- Obsidian 노트 자동 분류 (태그 자동 생성)

### Vercel 심화 | 2-3주
**목표:** 환경 변수, 데이터베이스 연결, API 라우트 최적화

## 💡 실제 학습 전략

### 1. 병렬 학습
TypeScript + Python을 동시에 진행하면 한 언어의 개념을 다른 언어로 적용하면서 깊이있게 배울 수 있어요.

### 2. 프로젝트 기반 학습
각 기술을 배운 후 **즉시 블로그 프로젝트에 적용**합니다.
- TypeScript 심화 → 더 복잡한 타입 정의
- Python → Obsidian 자동화 스크립트
- AI/ML → 블로그 추천 시스템

### 3. 지식 기록
배운 내용을 Obsidian wiki에 정리하고, 3개월마다 블로그 포스트로 변환합니다.

### 4. 검토 주기
- 주간: 학습 진행도 체크
- 월간: 예정한 내용 완료 여부 확인
- 분기별: 다음 학습 계획 수립

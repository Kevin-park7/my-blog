# Next.js로 블로그 만들기 #1: 프로젝트 셋업

개발자라면 자신의 기술 블로그가 필요합니다. 이 시리즈에서는 Next.js 14를 사용해 모던한 블로그를 만드는 방법을 알려드립니다.

## 왜 Next.js인가?

Next.js는 블로깅에 최적화된 프레임워크입니다:

- **빠른 성능**: Static Site Generation(SSG)으로 초고속 페이지 로딩
- **SEO 친화적**: 자동 메타데이터 관리, Open Graph 지원
- **개발 경험**: 핫 리로딩, 파일 기반 라우팅
- **확장성**: 앞으로 댓글, 구독, 분석 기능 추가 가능

## 프로젝트 셋업

### 1단계: Next.js 프로젝트 초기화

```bash
npx create-next-app@latest my-blog \
  --typescript \
  --tailwind \
  --app
```

이 명령은 다음을 설정합니다:
- TypeScript: 타입 안전성
- Tailwind CSS: 빠른 스타일링
- App Router: 최신 Next.js 구조

### 2단계: 필수 의존성 설치

```bash
cd my-blog
npm install
npm install contentlayer next-contentlayer
npm install date-fns
```

각 패키지의 역할:
- `contentlayer`: 마크다운 파일을 JSON으로 변환
- `date-fns`: 날짜 포맷팅

### 3단계: Contentlayer 설정

`contentlayer.config.ts` 파일을 프로젝트 루트에 생성합니다:

```typescript
import { defineDocumentType, makeSource } from 'contentlayer/source-files'

export const Post = defineDocumentType(() => ({
  name: 'Post',
  filePathPattern: `posts/**/*.md`,
  fields: {
    title: { type: 'string', required: true },
    date: { type: 'string', required: true },
    tags: { type: 'list', of: { type: 'string' } },
    series: { type: 'string' },
    seriesOrder: { type: 'number' },
  },
  computedFields: {
    slug: {
      type: 'string',
      resolve: (doc) => doc._raw.sourceFileName.replace(/\.md$/, ''),
    },
  },
}))

export default makeSource({
  contentDirPath: 'content/posts',
  documentTypes: [Post],
})
```

### 4단계: next.config.js 업데이트

```javascript
const { withContentlayer } = require('next-contentlayer')

const nextConfig = {
  pageExtensions: ['ts', 'tsx', 'js', 'jsx', 'md', 'mdx'],
}

module.exports = withContentlayer(nextConfig)
```

### 5단계: 폴더 구조 생성

```bash
mkdir -p content/posts
mkdir -p src/app/blog
mkdir -p src/components
```

폴더 구조:
```
my-blog/
├── content/posts/          # 마크다운 파일 저장
├── src/
│   ├── app/
│   │   ├── layout.tsx
│   │   ├── page.tsx
│   │   └── blog/           # 블로그 페이지
│   ├── components/         # 재사용 컴포넌트
│   └── lib/               # 유틸 함수
├── contentlayer.config.ts
└── next.config.js
```

## 개발 환경 확인

```bash
npm run dev
```

브라우저에서 `http://localhost:3000`을 열어 프로젝트가 정상 작동하는지 확인합니다.

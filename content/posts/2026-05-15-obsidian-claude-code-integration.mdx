# Claude Code와 Obsidian을 함께 쓰면 뭐가 좋을까?

> **목표 독자:** Claude Code를 시작한 지 얼마 안 되는 개발자
> **읽는 시간:** 8분
> **난이도:** 초급

---

## 들어가며

Claude Code를 쓸 때 이런 고민 없었나요?

- "내가 물어본 질문과 답변을 어디에 저장할까?"
- "배운 기술을 다시 물어보지 않으려면?"
- "AI와의 대화를 블로그 글로 만들 수 있을까?"

나도 처음엔 헷갈렸어요. Claude Code의 자동화는 편하지만, **내가 배운 걸 정리할 곳이 없었거든요.**

그래서 시작한 게 **Obsidian 연동**입니다. 단순한 메모장이 아니라, **Claude Code와 협력하는 지식 시스템**으로요.

---

## 그래서 뭐가 좋은데?

### 1️⃣ 배운 걸 두 번 물어보지 않아도 된다

**Before (Claude Code만):**
```
1주일 뒤...
나: "React의 의존성 배열이 뭐였더라?"
Claude: [또 설명함]
(시간 낭비...)
```

**After (Obsidian + Claude Code):**
```
1주일 뒤...
나: /query React 의존성 배열
Obsidian: 이미 저장된 내용 + 링크
(3초면 끝)
```

### 2️⃣ 블로그 콘텐츠가 자동으로 모인다

Claude와의 대화 → Obsidian 정리 → 블로그 발행

```
[GitHub PR의 설명](link) ✍️
    ↓
[Obsidian wiki](link) 📚
    ↓
[블로그 포스트](link) 📖
```

3달 뒤엔 블로그 포스트 5개가 준비되어 있어요.

### 3️⃣ 반복 학습이 자동으로 생긴다

같은 주제의 Q&A가 모이면, 패턴이 보여요.

- "React Hooks" 질문 5개 → 완벽 가이드 1개
- "Obsidian 팁" 질문 3개 → 입문 가이드 1개

**근본 원인:** 다시 물어본다는 건, 아직 확실하지 않다는 뜻이거든요.

---

## 실제로 어떻게 쓸까?

### 단계 1: Claude와 대화 (Raw 저장)

```
/ingest articles React Hook 의존성 배열 완벽 가이드
```

Claude: 원본을 Obsidian `raw/articles/` 폴더에 저장

### 단계 2: 정리 (Wiki 작성)

```
/query React Hook 의존성 배열
```

Claude: 관련 내용을 찾아 wiki 페이지 생성 또는 강화

### 단계 3: 출판 (Output 준비)

문서 옮김: `wiki/learning/` → `output/blog/`

## 켈빈의 실제 설정

### 폴더 구조

```
dev-vault/
├─ raw/              # 불변 원본
│  ├─ articles/      # 기사, 블로그 (수정 금지)
│  ├─ papers/        # 논문, 문서
│  ├─ code-snippets/ # 코드 예시
│  └─ sessions/      # Q&A 세션 기록 ← 새로 추가!
│
├─ wiki/             # AI가 정리한 콘텐츠
│  ├─ entities/      # 개념, 기술, 용어
│  ├─ learning/      # 학습 기록
│  │  └─ prompt-qa/  # Q&A 정리 ← 새로 추가!
│  ├─ blog-ideas/    # 블로그 아이디어
│  ├─ projects/      # 프로젝트 추적
│  └─ synthesis/     # 종합 분석
│
└─ output/           # 최종 결과물
   ├─ blog/          # 블로그 포스트
   ├─ apps/          # 앱 문서
   └─ guides/        # 가이드 ← 새로 추가!
```

### Q&A 저장 흐름

```
내 질문 + Claude 답변
    ↓
raw/sessions/2026-05-13-topic.md
(원본: Q&A 전체 보존)
    ↓
wiki/learning/prompt-qa/topic-qa.md
(정리: 핵심 + 학습 포인트)
    ↓
output/guides/topic-guide.md
(발행: 블로그 형식)
```

## 나의 3개 스킬

Obsidian을 Claude Code와 연동하기 위해 3개 스킬을 만들었어요:

### `/ingest` - 자료 저장
새로운 기술 자료를 raw/에 저장하고, wiki에 정리 페이지를 만들어줍니다.

```
/ingest articles REST API vs GraphQL 비교
```

### `/query` - 지식 검색
wiki에서 관련 내용을 찾아 종합 답변을 만들어줍니다.

```
/query GraphQL의 장점은?
```

### `/lint` - 정합성 점검
월 1회 실행해서 오래된 정보, 깨진 링크, 중복을 찾아줍니다.

```
/lint
```

### 3단계: 월 1회 정리하기
- `/lint`로 상태 점검
- 관련 Q&A 모아서 wiki 페이지 생성

## 3개월 뒤엔?

지금부터 매주 1개씩 Q&A를 저장하면:

- **1개월 뒤:** Q&A 4개 저장
